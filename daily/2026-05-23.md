## 📝 学习笔记

### AI × Web3 最小交叉流程图

把 Week 1 所有概念串了一遍，核心链路是：

**用户意图 → LLM 推理 → RAG 检索补充知识 → Agent ReAct 循环（Think/Act/Observe）→ 人工复核 → 钱包签名 → 链上执行 → 结果验证 → 返回 Agent**

几个核心认知：
- **边界最重要**：Agent 负责生成方案，人负责关键决策，链负责执行和记录——三方各管边界内的事
- **两处红线**：① Agent 生成代码/方案后必须人工复核 ② 链上操作必须用户手动签名，这两步不能跳过
- **RAG 不是万能**：知识库不维护，RAG 只会把旧错误送得更稳——chunk 质量 > chunk 数量
- **Context 是资源**：工具调用历史 + RAG chunk 都在占 context window，塞满了 Agent 会"忘事"

## 📮 打卡内容草稿

**Week 1 综合任务打卡：AI × Web3 最小交叉流程图（30 pts）**

画了一张从「用户发起任务」到「链上执行验证」的完整流程图，把 Week 1 学的 LLM、Prompt、Context、RAG、Agent、钱包、合约这些概念串成了一条链路。

**核心链路：**
用户自然语言输入 → LLM 推理 → RAG 检索补外部知识 → Agent ReAct 循环（Think/Act/Observe）→ 🔴 人工复核代码/参数 → 🔴 钱包签名授权 → 链上广播执行 → 区块确认 → Event 日志 + 状态验证 → 返回 Agent

**6 个关键标注：**
- 谁发起：用户（自然语言）
- 谁执行：Agent（推理+工具调用）+ 链（合约执行）
- 签名/付款：MetaMask 弹窗，用户手动确认
- 人工确认：两处红线（代码复核 + 钱包签名）
- 结果验证：tx hash + Event 日志 + Agent 自检
- 主要风险：Agent 幻觉 / RAG 检索不准 / 跳过人工确认 / 私钥泄露

**一句话总结：** AI 负责理解和生成，人负责决策和签名，链负责执行和记录——三方各守边界，才是可信的 AI × Web3 系统。
