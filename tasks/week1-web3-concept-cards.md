# Web3 基础概念卡片

> 8 个概念，每个用自己的话 + 后端类比。基于 Handbook 阅读 + Web3 运行原理直播笔记。

---

## 1. 私钥 / 助记词

**是什么**：私钥是你控制链上资产的唯一凭证。助记词是私钥的"人类可读版"，12 或 24 个英文单词，能还原出私钥。

**类比（后端）**：私钥 = 数据库 root 密码。谁拿到谁就能操作你的全部资产。没有"找回密码"功能——链不认识你是谁，只认签名。

**安全规则（血的教训）**：
- 私钥丢了无法找回，没有客服没有重置
- 泄露就等于丢资产，不需要对方"转走"，有你私钥就能签交易
- 严禁：截图保存、网盘备份、微信转发、复制粘贴到不明网站
- 主流被盗原因：助记词泄露、钓鱼网站授权、设备中木马

**为什么特别谨慎**：区块链不认实名身份，只认私钥签名。你在银行被盗还能冻结，链上交易不可逆——出去了就回不来。

---

## 2. EOA（外部账户）

**是什么**：以太坊最基础的钱包类型。一个私钥对应一个地址，每次操作都需手动签名。

**类比（后端）**：EOA = 裸数据库连接。私钥就是连接串，丢了全完。每次操作弹确认框 = 每次数据库操作都弹 sudo 密码。

**特点**：
- 私钥控制一切，没有恢复机制
- 每笔交易自己付 gas
- 不能写逻辑——转账就是转账，不能加"每天限额 0.1 ETH"

**痛点**：我用 MetaMask 的体会——每次都得点确认，切个网络还得手动加。对普通用户来说太反人类了。

---

## 3. AA（账户抽象 / Account Abstraction）

**是什么**：把钱包从"私钥 = 控制权"升级成"可编程账户"。钱包不再是简单的密钥对，而是链上一个智能合约，可以写规则。

**类比（后端）**：AA = 给数据库连接加个中间件层。可以写规则："每天转账上限 0.1 ETH，超过拒绝"、"超过 500U 需要两个管理员确认"。

**解决的问题**：
- 私钥丢了能通过守护者恢复（不用靠助记词赌命）
- 可以用 USDC 付 gas，不用非得有 ETH
- 多签：大额需要多人确认
- 能写交易规则：限额、白名单、时间锁

**跟我的关系**：后续如果做 Agent Wallet，AA 是基础——要能让 Agent 在有限权限下操作钱包。Agent 用 AA 钱包的核心价值是可以设**权限边界**——只允许 Agent 操作特定代币、设转账限额、加时间锁，防止 Agent 出 bug 或失控时造成损失。

---

## 4. 智能合约（Smart Contract）

**是什么**：跑在区块链上的代码。用 Solidity 写，部署到 EVM 上执行。一旦部署，代码不可改（除非设计了升级逻辑）。

**类比（后端）**：智能合约 = 永远在线的 API 服务，但有几个区别：
- 部署了就改不了（immutable）
- 每步执行都要付 gas
- 任何人都能调用，不需要 API key

**一笔合约调用的完整流程**：
1. 前端读钱包地址和当前网络
2. 前端根据 ABI 编码调用数据（比如 `vote(proposalId)`）
3. 钱包展示交易信息，用户确认签名
4. 交易广播到 RPC 节点
5. 验证者打包进区块
6. EVM 执行合约逻辑 → 成功更新状态 + emit event，失败 revert
7. 前端监听交易回执，更新页面
8. 索引器读 event，更新历史记录/看板

---

## 5. ABI（应用二进制接口）

**是什么**：合约的结构说明书。告诉前端："这个合约有哪些方法、输入什么参数、返回什么类型"。

**类比（后端）**：ABI = API 文档 / Swagger / OpenAPI spec。没 ABI，前端不知道怎么调合约。

**例子**：
```
{
  "name": "transfer",
  "inputs": [
    {"name": "to", "type": "address"},
    {"name": "amount", "type": "uint256"}
  ],
  "outputs": [{"name": "", "type": "bool"}]
}
```
→ 告诉前端：调 transfer 方法，传 to (地址) 和 amount (数量)，返回 bool。

## 6. Gas

**是什么**：在以太坊上执行任何操作都要付的费用。Gas = 计算资源的价格，防止无限循环和垃圾交易。

**类比（后端）**：Gas = 云服务的按量计费。但更狠——不光部署时付，每次调用都要付。而且不管你写得多烂，gas 照收不误。

**关键规则**：
- 读操作（view 函数）不收 gas
- 写操作（改状态）收 gas
- 存储写入贵（SSTORE），读取便宜（SLOAD）
- 循环写得烂 = gas 爆炸 = 交易失败但还收你钱

**写合约要注意的**：
- Solidity 里变量存 storage（永久，贵）还是 memory（临时，便宜）——搞错一个是 bug 也是浪费 gas
- 优化 gas 不是"可选"的，是必须的

**L2 补充**：
- 实际开发基本都在 L2（如 Arbitrum、Base）上，gas 比 L1 便宜几十到几百倍，主网主要用来结算。

## 7. Event（事件）

**是什么**：合约执行时发出的链上日志。前端和索引器可以监听这些日志，更新 UI 或统计数据。

**类比（后端）**：Event = 消息队列 / Webhook。合约做完事发个通知："转账完成了，接收人 0x...，数量 100"。

**怎么用**：
- 合约里 `emit Transfer(from, to, amount)`
- 前端监听这个 event，转账成功就弹通知
- 索引器（如 The Graph）扫描 event，建查询数据库
- indexed 关键字**：emit 时加了 `indexed` 的字段才能被高效检索，比如 `event Transfer(address indexed from, address indexed to, uint256 amount)`。不加只能全量扫描。

## 8. ERC-4337（账户抽象标准）

**是什么**：不修改以太坊底层协议，而是加了一层 UserOperation 来实现账户抽象。

**核心角色**：
| 角色 | 干什么 |
|------|--------|
| UserOperation | 用户的意图，不是传统交易，是一种新的数据结构 |
| Bundler | 收集 UserOperation，打包成一笔交易发给 EntryPoint |
| EntryPoint | 统一处理所有智能账户交易的合约 |
| Paymaster | 可以帮用户付 gas，或让用户用其他代币付 |

**流程**：用户发起操作 → 打包成 UserOperation → 发给 Bundler → Bundler 调 EntryPoint → EntryPoint 调你的智能账户执行。

**为什么重要**：ERC-4337 让 AA 不用等以太坊底层升级就能用。现有的钱包（MetaMask 等）不需要改，新加的这层在上面跑。

## 安全提醒（Web3 黄金法则）

区块链安全跟传统 Web 安全不一样——你银行被盗有人兜底，链上被盗就没了。

- **私钥 / 助记词 = 你的全部资产**，不是密码，没有重置
- **签名授权**：你在 MetaMask 点的"确认"不只是确认，是授权执行——确认前看清楚在签什么
- **钓鱼授权**：不明网站让你 connect 钱包 + 确认签名 = 可能是在授权对方转走你的资产
- **不要把私钥/助记词截图、存网盘、发微信**——这些平台不是为私钥安全设计的
