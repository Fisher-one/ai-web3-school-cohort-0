## 二、AI 侧节点（从上到下）

| 顺序 | 节点名称 | 颜色 | 内容说明 |
|------|----------|------|----------|
| 0 | **用户输入**（框外顶部） | 黑色文字 | 起点，自然语言描述任务 |
| 1 | **Prompt 组装** | 紫色框 | 用户需求 + 输出格式 + 系统提示 + 边界约束 |
| 2 | **Context** | 灰色虚线框 | Memory（历史偏好）+ Session State（当前会话） |
| 3 | **LLM 推理/判断** | 绿色框 | 接收 Prompt + Context 注入，决定下一步 |
| 4a | **RAG 检索** | 橙色框 | LLM 判断需要查资料时触发 |
| 4b | **Knowledge Base** | 黄色框 | RAG 的数据来源，双向箭头（查询 ↔ 返回） |
| 4c | **直接生成** | 黑色文字 | LLM 判断不需要 RAG，直接生成结果 |
| 5 | **Evaluation / 自检** | 青色框 | 检查输出是否符合 Prompt 要求（格式/内容/边界） |
| 6 | **输出结果（执行方案）** | 黑色文字 | 位于 Agent 框下方，交给 Web3 侧 |

### AI 侧连接关系

```
用户输入
  ↓
Prompt 组装 ←── Context（Memory 注入，虚线）
  ↓
LLM ←→ Context（Session State 双向读写，虚线）
  ↓（需要查资料？）
  ├── 是 → RAG 检索 ←→ Knowledge Base → 结果回流 → LLM
  └── 否 → 直接生成
          ↓
      Evaluation / 自检
          ↓
      输出结果（执行方案）
```

## 三、Web3 侧节点（从上到下，全部垂直排列）

| 顺序 | 节点名称 | 颜色 | 内容说明 |
|------|----------|------|----------|
| 1 | **🔴 人工复核** | 红色框 | 检查方案 / 参数 / Gas 费 |
| 2 | **🔴 钱包签名** | 红色框 | MetaMask 弹窗，用户手动确认 |
| 3 | **RPC 广播 → mempool** | 蓝色框 | 交易广播上链 |
| 4 | **验证者打包 → 区块上链** | 深蓝框 | 验证者/矿工打包区块 |
| 5 | **EVM 执行** | 绿色框 | ✓ 成功 → 状态更新 + Event；✗ 失败 → revert（Gas 仍扣） |
| 6 | **区块浏览器** | 青色框 | 查询 tx hash / 交易状态 |

## 四、6 项必填标注（对照检查）

打开 `aiweb3.excalidraw`，逐条确认：

  - 位置：Web3 大框标题行右侧，红色文字

  - 位置：钱包签名框右侧，黄底红字小框

- [ ] **必须人工确认（红线①）**
  - 内容：`🙋 必须人工确认（红线①）  检查：方案 / 参数 / Gas 费`
  - 位置：人工复核框右侧，黄底红字小框

- [ ] **如何验证结果**
  - 内容：`✅ 如何验证结果  tx hash + Event 日志 + Agent 自检`
  - 位置：区块浏览器框右侧，青底小框

- [ ] **风险点清单**（图左侧，红色面板，5 条）
  1. LLM 幻觉：生成错误代码 / 参数（AI侧）
  2. RAG 检索不准：知识库过期 → 旧错误被放大（AI侧）
  3. 人工复核形同虚设：没看清楚就确认（交界点）
  4. 交易不可逆：上链后无法撤回（Web3侧）
  5. 私钥泄露 / 盲签：不看弹窗就签名（Web3侧）

## 五、一句话总结

> AI 负责理解和生成，人负责决策和签名，链负责执行和记录——三方各守边界，才是可信的 AI × Web3 系统。
